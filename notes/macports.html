Commonly Used Ports (x86 / M1)

  1. ccache (compiler cache): https://ccache.dev
  2. dnscyrpt-proxy (encrypted dns): https://github.com/DNSCrypt/dnscrypt-proxy
  3. dosbox-x (enhanced dosbox): https://dosbox-x.com/
  4. gnupg2 (GNU pretty good privacy): https://www.gnupg.org/
  5. innoextract (extract files from Windows installers): https://constexpr.org/innoextract/
  6. libpst (extract files from Outlook PST archives): https://www.five-ten-sg.com/libpst/
  7. ncftp (FTP client): http://www.ncftpd.com/ncftp/
  8. newsboat (terminal based RSS reader): https://newsboat.org/
  9. octave (matlab-like math package): https://www.gnu.org/software/octave/
  10. openssh: https://www.openssh.com/
  11. qemu (multi-architecture emulator): https://www.qemu.org/
  12. rclone (sync files to cloud services): https://rclone.org/
  13. shellcheck (check the syntax of shell scripts): https://www.shellcheck.net/
  14. smartmontools (get SMART status for drives): https://www.smartmontools.org/
  15. tig (terminal based git history browser): https://jonas.github.io/tig/
  16. unshield (extract Installshield archives): https://github.com/twogood/unshield
  17. youtube-dl (video downloader): https://www.youtube-dl.org/

Contributing to MacPorts

  1. Create a fork on GitHub

  2. Clone the fork locally: 

    $ git clone https://github.com/username/macports-ports.git
    where username your github username

  3. Add the the official repository as the upstream: 
    $ git remote add upstream https://github.com/macports/macports-ports.git

  4. On GitHub, update the fork to include any upstream changes.

  5. Fetch upstream changes locally: 
    $ git pull
$ git fetch upstream

  6. Create a branch for changes: 

    $ git checkout -b portname-version
    where portname is the name of the port and version is the version of the port

  7. After making changes, lint the Portfile: 

    $ port lint --nitpick -D .

  8. Commit the changes: 

    $ git commit -a -m "portname: update to version" [-S]
    where portname is the name of the port, version is the version of the port, and -S is optional (assuming that signing commits is enabled)

  9. Push the changes upstream: 

    $ git push --set-upstream origin portname-version
    where portname is the name of the port and version is the version of the port

  10. Open a pull request on GitHub

  11. If changes are requested, make the changes, lint the Portfile (see above), and then override the prior commit: 

    $ git commit --amend -a -m "portname: update to version" [-S]
$ git push origin -f
    where portname is the name of the port, version is the version of the port, and -S is optional (assuming that signing commits is enabled)

  12. Switch back to the main/master branch: 

    $ git checkout master
